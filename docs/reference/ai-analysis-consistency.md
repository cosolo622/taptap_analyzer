## 一、Prompt模板设计

### 核心Prompt

```
你是一个专业的用户运营分析师，熟悉鹅鸭杀游戏。

请分析以下TapTap评价，输出：

1. 情感倾向：正向/负向/中性
   - 正向：明确表达满意、推荐
   - 负向：明确表达不满、批评、提到具体问题
   - 中性：客观陈述，无明显情感倾向

2. 问题分类：大类-细分问题格式
   分类体系：
   - 环境问题：低素质玩家、路人局体验差、语言攻击、村规问题、场外现象
   - 技术问题：优化不足、网络波动、闪退、发热严重、服务器不稳定
   - 商业化问题：氪金点过多、皮肤定价、福利少
   - 平衡性问题：角色强度不均、阵营优势过大
   - 内容问题：不耐玩、角色太少、地图太少
   - 匹配问题：水平差距大、段位匹配不合理
   - 功能建议：排位赛模式、新手引导、举报系统
   - 玩法建议：排位机制、角色机制
   
   注意：正向评价可能无问题分类，输出null

3. 一句话总结：核心内容概括（20字以内）

评价内容：
{content}

星级：{rating}

请以JSON格式输出：
{"sentiment": "", "problem_category": "", "summary": ""}
```

## 三、一致性保证机制

| 机制 | 说明 | 实现方式 |
|------|------|---------|
| **Prompt固化** | 使用固定的Prompt模板 | 代码中硬编码，版本控制 |
| **低温度参数** | temperature=0.3，减少随机性 | API调用时设置 |
| **Few-shot示例** | 在Prompt中加入典型示例 | 引导模型输出格式 |
| **后处理校验** | 对输出进行格式校验 | JSON解析+字段校验 |
| **人工抽检** | 定期抽检分析结果 | 每周抽检10条，调整Prompt |
| **版本记录** | 记录Prompt版本和效果 | 本文档 |

## 五、错误处理

### 常见错误及处理

| 错误类型 | 处理方式 |
|---------|---------|
| JSON解析失败 | 正则提取或重新调用API |
| 字段缺失 | 填充默认值或重新分析 |
| 分类不在体系内 | 归类到"其他"或人工审核 |
| API限流 | 指数退避重试 |

### 重试机制

```python
def analyze_with_retry(content, rating, max_retries=3):
    """
    带重试的分析函数
    
    Args:
        content: 评价内容
        rating: 星级
        max_retries: 最大重试次数
    
    Returns:
        dict: 分析结果
    
    替代方案：
    - 可以使用tenacity库实现更复杂的重试逻辑
    """
    for i in range(max_retries):
        try:
            result = analyze_review(content, rating)
            if validate_result(result):
                return result
        except Exception as e:
            wait_time = 2 ** i  # 指数退避
            time.sleep(wait_time)
    return get_default_result()
```
