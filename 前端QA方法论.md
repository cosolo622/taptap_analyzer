# 前端自动化QA方法论

## 核心理念
**任何前端修改完成后，必须使用Playwright MCP进行自动化验证，确保功能正常后再交付用户。**

## 方法论步骤

### 1. 导航与加载
- 使用 `playwright_navigate` 访问目标页面
- 使用 `waitUntil: networkidle` 等待网络空闲
- 用 `evaluate` 确认页面加载完成（检查title、body长度）

### 2. 功能测试
- 使用 `playwright_click` 点击菜单/按钮
- 使用 `evaluate` 获取页面内容验证显示正确
- 检查数据是否为有效值（不是0、undefined、null、空字符串）

### 3. 错误检测
- 使用 `console_logs` 检查控制台错误
- 重点关注：422/500错误、API调用失败、组件报错
- 使用 `playwright_get` 直接测试后端API返回

### 4. 截图存证
- 使用 `playwright_screenshot` 记录测试结果
- 便于问题定位和交付确认

## 必检清单
- [ ] 页面能正常加载
- [ ] 数据显示正确（非空值）
- [ ] 控制台无错误
- [ ] API返回正常
- [ ] 交互功能正常

## 常见陷阱
1. 组件不支持某些数据类型（如el-statistic只支持数字）
2. API参数未设默认值导致必填报错
3. 路由冲突导致请求被错误处理
4. 异步数据未等待加载完成

## 执行时机
- 完成前端开发后
- 告诉用户"修复完成"前
- 用户反馈问题时
- 验证bug修复后

## 核心原则
> **永远不要在没有自测的情况下告诉用户"完成了"**
