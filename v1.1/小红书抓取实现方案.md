1) 落地方案框架（本机稳定态 + 未来可转云）

- 定位 ：这是“浏览器自动化采集任务”，不是传统HTTP爬虫。
- 最优技术选型 ：
  - 执行层：Playwright（持久化用户目录）
  - 编排层：本机定时任务（后续可迁云调度）
  - 数据层：postgre sql（先本机），后续可接云库
  - 告警层：登录失效关键字检测 + 通知（企业微信/飞书）
- 核心状态机 ：
  - INIT -> AUTH_CHECK -> SEARCH_FILTER -> COLLECT -> VERIFY -> DONE
  - 任一步命中“登录后查看搜索结果” => AUTH_REQUIRED （人工补登）
- 为什么这样做 ：
  - 你已验证“同会话稳定、重启后易失效”，所以必须围绕 会话复用 设计
  - 站点限制关键是登录态/风控，不是页面解析复杂度

sync_playwright
from datetime import datetime

KEYWORD = "鹅鸭杀"

search_result?keyword={KEYWORD}"

def to_num(s: str) -> int:
    s = (s or "").replace("赞", "").
    strip()
    if not s:
        return 0
    if "亿" in s:
        return int(float(s.replace("
        亿","")) * 1e8)
    if "万" in s:
        return int(float(s.replace("
        万","")) * 1e4)
    return int(float(s.replace("+",
    "")))

with sync_playwright() as p:
    context = p.chromium.
    launch_persistent_context(
        user_data_dir=r"D:\xhs_profil
        e",
        headless=False
    )
    page = context.new_page()
    page.goto(URL, 
    wait_until="domcontentloaded")

    if LOGIN_GATE_TEXT in page.
    inner_text("body"):
        raise RuntimeError
        ("AUTH_REQUIRED")

    page.get_by_text("筛选", 
    exact=True).click()
    page.get_by_text("一天内", 
    exact=True).click()
    page.get_by_text("收起", 
    exact=True).click()

    # 采集
    items = []
    cards = page.query_selector_all
    ("section.note-item")
    for c in cards:
        title = (c.query_selector("a.
        title") or c).inner_text().
        split("\n")[0].strip()
        time_txt = (c.query_selector
        (".time").inner_text() if c.
        query_selector(".time") else 
        "")
        like_txt = (c.query_selector
        (".count").inner_text() if c.
        query_selector(".count") 
        else "0")
        items.append({
            "title": title,
            "date": time_txt,
            "likes": like_txt,
            "likes_num": to_num
            (like_txt),
            "ts": datetime.now().
            isoformat
            (timespec="seconds")
        })

    items.sort(key=lambda x: x
    ["likes_num"], reverse=True)
    print(items[:20])

2) 本轮压测方式、过程、结果（给后续模型的明确口径）

- 压测目标 ：模拟“每隔N小时一次较彻底探查”是否容易触发风控
- 执行口径（严格） ：
  - 登录后进入小红书搜索“鹅鸭杀”
  - 筛选： 综合 + 一天内
  - 至少20条：逐条“点进详情 -> 快速返回”，不做正文阅读
  - 采集字段：标题、点赞、发帖时间
  - 输出：按点赞降序
- 压测结果 ：
  - 已完成20条点进点出和批量采集
  - 本轮结束后未触发登录拦截，未出现 300012 / IP风险 / 安全限制
  - 说明：当前频率与动作在本机会话下可行
必须强调给后续模型的三点

- 这不是“裸请求爬虫”，而是 有登录态的浏览器自动化采集流程
- 先做登录态探测，再做筛选与采集；登录失效时进入人工补登分支
- 目标是稳定运行，不是高频爆量抓取；风控优先级高于吞吐
