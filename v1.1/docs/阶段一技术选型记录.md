## 一、项目概述

### 1.1 项目目标
构建一个多平台产品舆情监控系统，支持：
- TapTap 评价监控
- 小红书舆情监控（后续扩展）
- 其他平台（微博等）预留扩展能力

### 1.2 数据规模预估

| 产品 | 当前数据量 | 预估年增长 | 总量预估 |
|-----|-----------|-----------|---------|
| 王者荣耀 | 36万条 | +10万/年 | 46万+ |
| 鹅鸭杀 | 8千条 | +3万/年 | 4万+ |
| 小红书 | 0 | +30万/年 | 30万+ |
| **合计** | **~37万** | **~43万/年** | **80万+/年** |

**三年预估**：200-300万条评价数据

## 四、数据库技术选型

### 4.1 选型对比

| 数据库 | 类型 | 适用场景 | 本项目适用性 |
|-------|------|---------|------------|
| **SQLite** | 嵌入式关系型 | 本地开发、小数据量 | ⭐⭐⭐ 够用但受限 |
| **PostgreSQL** | 关系型 | 生产环境、中大数据量 | ⭐⭐⭐⭐⭐ 推荐 |
| **ClickHouse** | OLAP 分析型 | 大数据分析、聚合查询 | ⭐⭐⭐⭐ 可选加速 |
| **MongoDB** | 文档型 | 非结构化数据 | ⭐⭐ 非必须 |
| **Elasticsearch** | 搜索引擎 | 全文搜索 | ⭐⭐ 非必须 |
| **Trino** | 分布式查询引擎 | 跨源分析、PB级数据 | ⭐ 过度设计 |

### 4.3 选型理由

#### 为什么选 PostgreSQL？

| 维度 | PostgreSQL 优势 |
|-----|----------------|
| **数据规模** | 1000万+ 数据完全没问题 |
| **写入性能** | 1-5万条/秒，爬虫写入绰绰有余 |
| **查询性能** | 合理建索引，复杂查询秒级响应 |
| **事务支持** | ACID 事务，数据安全 |
| **扩展性** | 支持分区表、全文搜索扩展 |
| **生态** | SQLAlchemy ORM 完美支持 |
| **迁移成本** | 本地 PostgreSQL → 云 RDS 零代码改动 |

#### 为什么不用其他方案？

| 方案 | 不选原因 |
|-----|---------|
| SQLite | 并发写入受限、生产环境不推荐 |
| MySQL | PostgreSQL 功能更丰富（JSON、数组类型等） |
| MongoDB | 关系型数据更适合 SQL |
| Trino | 过度设计，数据量未达到 PB 级 |

#### 什么时候加 ClickHouse？

| 数据量 | 建议 |
|-------|-----|
| < 50万 | PostgreSQL 足够 |
| 50-100万 | 可考虑加 ClickHouse |
| > 100万 | 建议加 ClickHouse 加速分析 |

## 五、爬虫技术选型

### 5.1 技术栈

| 技术 | 选型理由 |
|-----|---------|
| **Scrapling** | 反爬绕过、自适应解析、断点续爬、并发支持 |
| **APScheduler** | 定时任务调度 |

### 5.2 Scrapling 核心优势

| 特性 | 说明 | 对项目的帮助 |
|-----|-----|------------|
| 反爬绕过 | 自动绕过 Cloudflare、验证码 | TapTap/小红书可能有反爬 |
| 自适应解析 | 页面结构变化自动适应 | 减少维护成本 |
| 断点续爬 | 支持 pause/resume | 大规模爬取更稳定 |
| 并发爬取 | 内置并发和代理轮换 | 提升爬取效率 |
| 多种模式 | get/fetch/stealthy-fetch | 按需选择，灵活应对 |

### 5.3 爬虫策略

| 平台 | 爬取方式 | 频率 | 备注 |
|-----|---------|-----|-----|
| TapTap | stealthy-fetch | 每日增量 | 可能有反爬 |
| 小红书 | stealthy-fetch | 每日增量 | 反爬较强，需研究 |

## 六、数据模型设计

### 6.1 核心表结构

```sql
-- 产品表
CREATE TABLE products (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,           -- 产品名称：鹅鸭杀
    code VARCHAR(50) UNIQUE,              -- 产品代码：goose_goose_duck
    created_at TIMESTAMP DEFAULT NOW()
);

-- 平台表
CREATE TABLE platforms (
    id SERIAL PRIMARY KEY,
    name VARCHAR(50) NOT NULL,            -- 平台名称：TapTap
    code VARCHAR(30) UNIQUE,              -- 平台代码：taptap
    base_url VARCHAR(200),                -- 基础URL
    created_at TIMESTAMP DEFAULT NOW()
);

-- 评价表（核心）
CREATE TABLE reviews (
    id SERIAL PRIMARY KEY,
    product_id INTEGER REFERENCES products(id),
    platform_id INTEGER REFERENCES platforms(id),
    review_id VARCHAR(100),               -- 平台评价ID
    content TEXT,                         -- 评价内容
    rating INTEGER,                       -- 星级
    sentiment VARCHAR(20),                -- 情感标签
    problem_category VARCHAR(100),        -- 问题分类
    review_date DATE,                     -- 评价日期
    crawl_date TIMESTAMP DEFAULT NOW(),   -- 爬取时间
    raw_data JSONB,                       -- 原始数据（备用）
    UNIQUE(platform_id, review_id)        -- 防重复
);

-- 爬取日志表
CREATE TABLE crawl_logs (
    id SERIAL PRIMARY KEY,
    spider_name VARCHAR(50),
    product_id INTEGER REFERENCES products(id),
    platform_id INTEGER REFERENCES platforms(id),
    start_time TIMESTAMP,
    end_time TIMESTAMP,
    pages_crawled INTEGER DEFAULT 0,
    reviews_added INTEGER DEFAULT 0,
    reviews_updated INTEGER DEFAULT 0,
    errors INTEGER DEFAULT 0,
    status VARCHAR(20)                    -- running/success/failed
);

-- 词云缓存表
CREATE TABLE wordcloud_cache (
    id SERIAL PRIMARY KEY,
    product_id INTEGER REFERENCES products(id),
    platform_id INTEGER REFERENCES platforms(id),
    week_key VARCHAR(20),                 -- 如：2025-W10
    image_path VARCHAR(500),              -- 词云图片路径
    top_words JSONB,                      -- 高频词列表
    created_at TIMESTAMP DEFAULT NOW()
);

-- 分析报告缓存表
CREATE TABLE report_cache (
    id SERIAL PRIMARY KEY,
    product_id INTEGER REFERENCES products(id),
    platform_id INTEGER REFERENCES platforms(id),
    date_range_start DATE,
    date_range_end DATE,
    report_content TEXT,                  -- LLM 生成的报告
    created_at TIMESTAMP DEFAULT NOW()
);

-- 预警规则表
CREATE TABLE alert_rules (
    id SERIAL PRIMARY KEY,
    product_id INTEGER REFERENCES products(id),
    platform_id INTEGER REFERENCES platforms(id),
    metric_type VARCHAR(50),              -- negative_count/negative_rate
    threshold DECIMAL(10,2),              -- 阈值
    time_window INTEGER,                  -- 时间窗口（天）
    is_enabled BOOLEAN DEFAULT TRUE,
    last_triggered TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 6.2 索引设计

```sql
-- 评价表索引
CREATE INDEX idx_reviews_product_platform ON reviews(product_id, platform_id);
CREATE INDEX idx_reviews_date ON reviews(review_date);
CREATE INDEX idx_reviews_sentiment ON reviews(sentiment);
CREATE INDEX idx_reviews_category ON reviews(problem_category);

-- 爬取日志索引
CREATE INDEX idx_crawl_logs_time ON crawl_logs(start_time);
```
